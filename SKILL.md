# 博客解说

深度解读热门内容，提取结构化方法论。

## 核心能力

- 将任意长视频/播客/博客内容转译为结构清晰的提炼报告
- 分析内容的叙事结构、表达技巧、观点框架
- 输出可直接用于内容创作的公式和模板
- 识别内容的传播要素和受众共鸣点
- **从视频到逐字稿的降维创作能力**（如7小时访谈→33分钟视频→15分钟逐字稿）

## 依赖工具

| 工具 | 用途 |
|---|---|
| `opencli-rs youtube transcript` | 获取YouTube视频字幕 |
| `browser` | 获取视频描述、页面内容 |
| `exec` | 运行命令、处理数据 |
| `feishu_create_doc` | 将解读结果推送至飞书云文档 |

## 执行流程

### 第一步：选题判断与信息收集

**第一步不是"获取内容"，是先找到核心问题。**

拿到素材，先问自己：

```
这个访谈/视频在回答一个什么问题？

How? / Why? / What? / What if?

这个问题能被普通读者关心吗？

这个问题有冲突感吗？
```

**案例：**
- Karpathy访谈 → 核心问题："一个最懂AI的人，为什么说AI让他精神错乱？"（What if?）
- 谢赛宁访谈 → 核心问题："一个普通人，是怎么走到AI舞台中央的？"（How?）

**然后才是获取内容、验证这个核心问题是否成立。**

---

**选题判断维度：**

| 维度 | 自检问题 | 例子 |
|---|---|---|
| **问题感** | 能用How/Why/What/What if表述吗？ | "AI让他精神错乱？" |
| **冲突感** | 有没有反直觉的点？ | "最懂AI的人停止写代码" |
| **好奇心** | 一句话能让人想知道更多吗？ | "普通人走到舞台中央？" |
| **重要性** | 读者为什么要care？ | 影响他们的工作/认知？ |

**选题 > 写作** — 如果核心问题不够有力，再好的文笔也救不回来。

**获取视频/播客的完整字幕：**

```bash
opencli-rs youtube transcript "[VIDEO_ID]" --format md
```

**获取视频描述信息（包含开场白、关键词、结构线索）：**

从YouTube页面提取视频描述，这是博主精心设计的"门面"，往往包含：
- 核心观点浓缩
- 关键词炸弹（用于标题设计的元素）
- 叙事线索引

### 第二步：结构分析

对照 `references/content-analysis-framework.md` 进行逐项分析：

1. **选题判断**：是否有冲突感？是否能引发好奇心？
2. **人物背景**：被采访者的权威性是否被建立？（公式：具体title + 关键成就 + 反差补充）
3. **开场结构**：悬念/反差设置、权威背书、信息量承诺
4. **展开结构**：三层递进（概念→理论→实例）、正反合对比
5. **表达技巧**：类比翻译、数据展示反差、节奏控制
6. **结尾结构**：金句收尾、总结压缩（4法则/4象限）
7. **标题设计**：关键词炸弹、观点型标题、人物背书
8. **降维逻辑**：从长视频到短内容的提炼路径

### 第三步：提炼公式

从分析结果中提取：

- 可套用的**框架模板**（3种类型）
- 可直接引用的**金句库**
- 内容生产的**检查清单**
- 专属**词汇转化表**（专业→通俗）
- **降维创作模板**（长视频→精华→逐字稿的路径）

### 第四步：输出

按以下格式整理，通过 `feishu_create_doc` 推送：

```
# 内容解读报告：[来源标题]

## 核心问题
[这个内容在回答什么问题？How / Why / What / What if]

## 基本信息
来源、人物、时长

## 选题分析
[核心问题是否有冲突感/好奇心/重要性]

## 人物背景
[如何建立权威性]

## 标题结构分析
[「关键词炸弹」模式解析]

## 开场结构分析
[分析结果]

## 内容展开逻辑
[结构图谱/要点——每个部分如何回答核心问题]

## 降维创作路径
[从原素材到成品的提炼逻辑]

## 核心方法提炼
[可复用的框架/公式]

## 金句库
[摘录]

## 传播要素分析
[为什么能火]

## 适用场景
[哪些创作可直接套用]
```

## 核心原则

- **问题 > 人物**：《写作是门手艺》核心——文章不是陈述事实，是回答一个问题。How? Why? What? What if? 人物只是答案的载体，问题才是灵魂
- **选题 > 写作**：先判断选题是否有冲突感，再动笔
- **框架是给自己的，输出是给读者的**：分析框架可以有结构，最终输出要像读书不像看大纲
- **不只总结，要解构**：揭示内容为什么有效，而非仅提炼要点
- **可操作，可复用**：每个分析都要产出可直接落地的创作公式
- **翻译为易懂语言**：把专业概念转化为创作者能用的白话表述
- **叙事优先于罗列**：全篇围绕一个核心主题，让每个段落回答"读者为什么在乎"
- **降维创作意识**：理解"海量素材→精华内容→不同形态成品"的提炼路径
